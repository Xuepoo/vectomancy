# 优化指南：避免数学软件崩溃

Vectomancy 将图像转换为数学方程。如果方程数量过多，GeoGebra、Kmplot 或 Desmos 等软件会因为内存不足或 UI 渲染过慢而崩溃。这种现象称为 AST 爆炸。

为了解决这个问题，请使用 `--tolerance` 和 `--min-path-len` 参数。

## 参数说明

### --tolerance

该参数控制路径简化的程度。值越大，方程越少，但精度会降低。建议设置为 2.0 到 5.0。

### --min-path-len

此参数用于忽略过短的路径。建议设置为 10。

## 建议

调整这些参数可以显著减少方程数量，确保软件运行流畅。如果软件仍然崩溃，请尝试增大这两个参数的值。
